装扮机器人V1.0开发记录

Bilibili 提供的一种个性化装饰，包括主题、表情包、空间背景等。
每一个装扮在购买时，都可以获取到唯一的编号。并且大部分装扮都是有库存的。

1 目标

  • 统计装扮的库存、编号信息
  • 统计和昨日环比销售增长情况
  • 自动统计在售装扮，新装扮不需要人工输入
  • 定时生成动态自动发布
  • 低成本托管部署

2 核心问题

从目标来看，其实要做的是一个爬虫 + 定时发布。

爬虫方面选择使用 python，因为其本身就有比较完备的生态链，也是目前主流的爬虫语言。
定时发布选择使用 github actions，之前试过使用其进行 B 站账号自动打卡升级，方案也比较成熟。

3 方案设计

3.1 爬虫

爬虫的思路其实很简单，就是 Get 网页/API，获取数据，然后整理后存储。

可惜目前开源的 API 里，没有装扮的 API 信息。比如说https://github.com/SocialSisterYi/bilibili-API-collect。但是其他的诸如动态的读取、发送之类的都有现成的，所以退而求次，抓包 + Fork 整起。

3.1.1 抓包

装扮页面在网页版上是无法打开的，会提示请打开客户端内打开。所以我是用安卓模拟器 + Fiddler 抓取的。但是后来发现，HTML5 版本也是可以打开的，所以只需要修改 UA，然后在 Chrome 上抓取即可。

3.1.2 模块开发

作为一个 python 初学者，站在了前人的基础上先做一个能跑起来的。所以使用的是https://github.com/chuiba/bilibili-api，在这个基础上，添加了新模块，用于封装装扮相关的 API。

3.1.3 统计功能开发

这里的逻辑就比较简单了，

  • 从装扮首页获取所有的装扮编号
  • 逐一读取装扮信息（如名称）、最新编号信息
  • 和昨日的进行比较
  • 将统计结果发动态
